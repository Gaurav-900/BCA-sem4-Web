📘 Unit I — OOP Concepts

Basic Concepts of Object Oriented Programming

Object-Oriented Programming (OOP) is a programming paradigm that organizes software design around data (objects) rather than functions and logic. Java is a purely object-oriented language that implements all core OOP principles.

1. Objects and Classes

Class

Definition

A class is a user-defined blueprint or prototype from which objects are created. It defines the properties (attributes) and behaviors (methods) that objects of that type will have. A class is a logical entity.

  • A class encapsulates data (fields) and methods that operate on that data.
  • Classes do not consume memory until objects are created from them.
  • Syntax: class ClassName { // fields and methods }

Object

Definition

An object is an instance of a class — a real-world entity that has state (attributes), behavior (methods), and identity (unique reference). Objects are physical entities that occupy memory.

  • State: Data stored in fields (e.g., name, age, color).
  • Behavior: Actions performed via methods (e.g., run(), eat()).
  • Identity: Unique memory address that differentiates one object from another.
Class and Object Example Java
// Class definition
class Car {
    // Fields (state)
    String brand;
    int speed;

    // Method (behavior)
    void accelerate() {
        speed += 10;
        System.out.println(brand + " is going at " + speed + " km/h");
    }
}

// Creating objects
public class Main {
    public static void main(String[] args) {
        Car c1 = new Car();  // Object creation
        c1.brand = "Toyota";
        c1.speed = 60;
        c1.accelerate();  // Output: Toyota is going at 70 km/h
    }
}

2. Data Abstraction & Encapsulation

Abstraction

Definition

Abstraction is the process of hiding internal implementation details and showing only the essential features of an object. It focuses on what an object does, not how it does it.

  • Reduces complexity by hiding unnecessary details from the user.
  • Achieved in Java via abstract classes and interfaces.
  • Real-world example: A car driver uses the steering wheel and pedals without knowing the engine internals.

Encapsulation

Definition

Encapsulation is the mechanism of bundling data (fields) and the methods that operate on that data into a single unit (class), and restricting direct access to some components using access modifiers.

  • Achieved using private fields with public getter/setter methods.
  • Protects data from accidental modification.
  • Also called data hiding.
Encapsulation Example Java
class BankAccount {
    private double balance;  // hidden from outside

    public void deposit(double amount) {
        if (amount > 0) balance += amount;
    }

    public double getBalance() {
        return balance;  // controlled access
    }
}

3. Inheritance

Definition

Inheritance is the mechanism by which one class (child/subclass) acquires the properties and behaviors of another class (parent/superclass). It represents an IS-A relationship.

  • Promotes code reusability — child classes inherit fields and methods.
  • Achieved using the extends keyword in Java.
  • Java supports single, multilevel, and hierarchical inheritance directly. Multiple inheritance is achieved via interfaces.
  • The parent class is also called base class or superclass; the child is called derived class or subclass.

4. Polymorphism

Definition

Polymorphism means "many forms." It allows one interface to be used for a general class of actions. The same method name can behave differently depending on the object or arguments.

Two types:

  • Compile-time Polymorphism (Static Binding): Method Overloading — multiple methods with the same name but different parameters.
  • Runtime Polymorphism (Dynamic Binding): Method Overriding — child class redefines a parent method; resolved at runtime.

5. Dynamic Binding

Definition

Dynamic Binding (late binding) is the process where the method call is resolved at runtime rather than compile time. Java resolves overridden method calls dynamically based on the actual object type.

Dynamic Binding Example Java
class Animal {
    void sound() { System.out.println("Generic sound"); }
}
class Dog extends Animal {
    void sound() { System.out.println("Woof!"); }
}
Animal a = new Dog();  // reference is Animal, object is Dog
a.sound();  // Output: Woof! — resolved at runtime (dynamic binding)

6. Message Passing

Definition

Message Passing is the process by which objects communicate with each other by invoking methods (sending messages). When an object calls a method of another object, it is said to be "sending a message."

  • A message consists of: the object name, the method name, and any parameters.
  • Example: obj.methodName(parameters);
  • Enables loose coupling between objects — each object handles its own data.

📘 Unit I — JVM & Architecture

Java Virtual Machine (JVM) & Byte Code

The JVM is the engine that runs Java bytecode. It provides a platform-independent execution environment — the key to Java's "Write Once, Run Anywhere" capability.

How Java Programs Execute

Execution Flow

Source Code (.java) → javac (compiler) → Bytecode (.class) → JVM (interpreter/JIT) → Machine Code → Output

JVM Architecture

Figure 1: JVM Architecture — Class Loader → Runtime Data Areas → Execution Engine

JVM Components Explained

1. Class Loader Subsystem

  • Loading: Reads .class files from disk/network.
  • Linking: Verifies bytecode, prepares memory, resolves symbolic references.
  • Initialization: Executes static initializers and static blocks.

2. Runtime Data Areas (Memory Areas)

AreaPer Thread?Stores
Method AreaSharedClass-level data, static variables, method bytecode
HeapSharedAll objects and arrays (garbage collected)
JVM StackPer ThreadStack frames for each method call, local variables, operands
PC RegisterPer ThreadAddress of the current executing bytecode instruction
Native Method StackPer ThreadNative (C/C++) method calls

3. Execution Engine

  • Interpreter: Reads and executes bytecode instructions one by one. Slower but starts quickly.
  • JIT (Just-In-Time) Compiler: Identifies "hot" code (frequently executed), compiles entire methods to native code for speed.
  • Garbage Collector: Automatically frees heap memory occupied by objects no longer reachable.

Bytecode Interpretation

Definition

Bytecode is an intermediate representation of Java programs — neither source code nor machine code. It is platform-neutral and is interpreted (or JIT-compiled) by the JVM on any host machine.

  • javac compiles .java source to .class (bytecode) files.
  • Bytecode is compact, approximately 10-20% of the source code size.
  • The JVM can verify bytecode before execution for security.
  • This is what makes Java platform independent.

Command Line Arguments

Definition

Command-line arguments are values passed to a program at the time of execution. In Java, they are received by the main method as a String[] array named args.

Command Line Arguments Example Java
public class CmdDemo {
    public static void main(String[] args) {
        System.out.println("Number of arguments: " + args.length);

        for (int i = 0; i < args.length; i++) {
            System.out.println("args[" + i + "] = " + args[i]);
        }

        // Converting string arg to int
        if (args.length > 0) {
            int num = Integer.parseInt(args[0]);
            System.out.println("Parsed number: " + num);
        }
    }
}
// Run: java CmdDemo 42 Hello World
// Output:
// Number of arguments: 3
// args[0] = 42
// args[1] = Hello
// args[2] = World
// Parsed number: 42
Important Notes
  • All command-line arguments are received as Strings.
  • To use numeric values, convert using Integer.parseInt(), Double.parseDouble(), etc.
  • args.length gives the number of arguments passed.
  • If no arguments are passed, args.length == 0 (not null).

Type Casting

Definition

Type Casting is the process of converting a value from one data type to another. Java supports two kinds:

Widening (Implicit) Casting

Converting a smaller type to a larger type automatically. No data loss. Done by the compiler.

byte → short → int → long → float → double
Widening Casting Java
int i = 100;
long l = i;       // automatic widening
double d = l;     // automatic widening
System.out.println(d);  // 100.0

Narrowing (Explicit) Casting

Converting a larger type to a smaller type. May cause data loss. Must be done explicitly by programmer.

Narrowing Casting Java
double d = 9.99;
int i = (int) d;   // explicit cast — fractional part truncated
System.out.println(i);  // 9 (not 10!)

long l = 300;
byte b = (byte) l;  // 300 overflows byte range (-128 to 127)
System.out.println(b);  // 44 (300 % 256 = 44)

📗 Unit II — Decision Making

Decision Making & Branching

Decision-making statements allow a program to choose a path of execution based on conditions. Java supports if, if-else, else-if ladder, and switch.

1. if Statement

Executes a block of code only if the condition is true.

if Statement Java
int marks = 75;
if (marks >= 50) {
    System.out.println("Pass");
}

2. if-else Statement

Executes one block if condition is true, another block if false.

if-else Java
int n = 7;
if (n % 2 == 0) {
    System.out.println("Even");
} else {
    System.out.println("Odd");   // Output: Odd
}

3. else-if Ladder

Tests multiple conditions in sequence. The first true condition executes; rest are skipped.

else-if Ladder — Grade Finder Java
int score = 82;
if (score >= 90) {
    System.out.println("Grade: A");
} else if (score >= 80) {
    System.out.println("Grade: B");  // Output: Grade: B
} else if (score >= 70) {
    System.out.println("Grade: C");
} else {
    System.out.println("Grade: F");
}

4. switch Statement

Definition

The switch statement evaluates an expression and transfers control to the matching case label. It is an efficient alternative to long else-if ladders when checking a single variable against multiple constant values.

  • Works with: byte, short, int, char, String, enum.
  • break prevents fall-through to the next case.
  • default executes when no case matches (optional but recommended).
  • Without break, execution falls through all subsequent cases.
switch Statement Java
int day = 3;
switch (day) {
    case 1:
        System.out.println("Monday"); break;
    case 2:
        System.out.println("Tuesday"); break;
    case 3:
        System.out.println("Wednesday"); break; // Output: Wednesday
    default:
        System.out.println("Invalid day");
}

📗 Unit II — Looping

Looping Statements

Looping allows executing a block of code repeatedly. Java provides for, while, do-while, and enhanced for-each loops.

1. for Loop

Used when the number of iterations is known in advance.

for (initialization; condition; update) { body }
for Loop Java
// Print 1 to 5
for (int i = 1; i <= 5; i++) {
    System.out.print(i + " ");
}
// Output: 1 2 3 4 5

// Multiplication table of 5
for (int i = 1; i <= 10; i++) {
    System.out.println("5 x " + i + " = " + (5 * i));
}

2. while Loop

Used when the number of iterations is not known in advance. Checks condition before executing the body.

while (condition) { body }
while Loop Java
int n = 1;
while (n <= 5) {
    System.out.print(n + " ");
    n++;
}
// Output: 1 2 3 4 5

// Sum of digits
int num = 12345, sum = 0;
while (num > 0) {
    sum += num % 10;
    num /= 10;
}
System.out.println("Sum of digits: " + sum);  // 15

3. do-while Loop

Executes the body at least once, then checks condition. Condition is checked after the body.

do { body } while (condition);
do-while Loop Java
int i = 1;
do {
    System.out.print(i + " ");
    i++;
} while (i <= 5);
// Output: 1 2 3 4 5

// Even if condition is false, executes once:
int x = 10;
do {
    System.out.println("Runs once");
} while (x < 5);  // condition false, but body runs once

4. Enhanced for Loop (for-each)

Simplified loop for iterating over arrays and collections.

for-each Loop Java
int[] nums = {10, 20, 30, 40, 50};
for (int n : nums) {
    System.out.print(n + " ");
}
// Output: 10 20 30 40 50

// break example

Definition

this is a reference to the current object. It is used to distinguish instance variables from method parameters when they share the same name, and to call other constructors.

Definition

Access modifiers control the visibility (accessibility) of classes, methods, and variables from other classes and packages.

A method can call another method of the same class directly (without creating an object, if both are non-static, or using this).

        System.out.println("In Method B");
    }

    public static void main(String[] args) {
        new Demo().methodA();
        // Output: In Method A
        //         In Method B
    }
}
📗 Unit II — Constructors & Wrapper Classes

Constructors & Wrapper Classes

Constructors

Definition

A constructor is a special method that is automatically called when an object is created. It has the same name as the class and no return type (not even void). Its purpose is to initialize an object.

Types of Constructors

All Constructor Types Java
class Employee {
    int id;
    String name;
    double salary;

    // 1. Default Constructor (no parameters)
    Employee() {
        id = 0; name = "Unknown"; salary = 0.0;
    }

    // 2. Parameterized Constructor
    Employee(int id, String name, double salary) {
        this.id     = id;
        this.name   = name;
        this.salary = salary;
    }

    // 3. Copy Constructor
    Employee(Employee other) {
        this.id     = other.id;
        this.name   = other.name;
        this.salary = other.salary;
    }

    // 4. Constructor Overloading (same name, different params)
    Employee(int id, String name) {
        this(id, name, 30000.0);  // calling another constructor
    }

    void display() {
        System.out.println(id + " | " + name + " | " + salary);
    }
}

// Usage

Polymorphism: Overriding, Overloading & final

Method Overloading (Compile-time Polymorphism)

Definition

Method Overloading is defining multiple methods in the same class with the same name but different parameter lists (different number, type, or order of parameters). Resolved at compile time.

Method Overloading Java
class Calculator {
    int    add(int a, int b)               { return a + b; }
    double add(double a, double b)         { return a + b; }
    int    add(int a, int b, int c)        { return a + b + c; }
    String add(String a, String b)         { return a + b; }
}

Calculator c = new Calculator();
c.add(2, 3);          // → 5 (int version called)
c.add(2.5, 1.5);      // → 4.0 (double version called)
c.add("Hi", " There"); // → "Hi There"

Method Overriding (Runtime Polymorphism)

Definition

Method Overriding occurs when a subclass redefines a method that is already defined in its superclass with the same name, return type, and parameters. Resolved at runtime based on the actual object type.

  • The overriding method cannot be more restrictive than the overridden method.
  • Use @Override annotation to signal intent (compiler checks).
  • Cannot override static, private, or final methods.
  • Use super.method() to call the parent's version.
Method Overriding Java
class Shape {
    void draw() { System.out.println("Drawing a shape"); }
}

class Circle extends Shape {
    @Override
    void draw() {
        super.draw();  // calls parent version first
        System.out.println("Drawing a circle");
    }
}

Shape s = new Circle();  // parent ref, child object
s.draw();
// Output: Drawing a shape
//         Drawing a circle

Abstract Classes & Interfaces

Abstract Classes

Definition

An abstract class is a class declared with the abstract keyword. It may contain abstract methods (without body) and concrete methods (with body). An abstract class cannot be instantiated directly.

  • Used to provide a common base with partial implementation.
  • Subclasses must implement all abstract methods (or be abstract themselves).
  • Can have constructors, instance variables, and concrete methods.
Abstract Class Example Java
abstract class Shape {
    String color;

    abstract double area();    // abstract: no body, must be overridden

    void displayColor() {      // concrete method
        System.out.println("Color: " + color);
    }
}

class Circle extends Shape {
    double radius;
    Circle(double r) { radius = r; color = "Red"; }

    @Override
    double area() { return Math.PI * radius * radius; }
}

// Shape s = new Shape();  // ERROR: cannot instantiate abstract class
Shape s = new Circle(5);
System.out.println(s.area());   // 78.54
s.displayColor();              // Color: Red

Interfaces

Definition

An interface is a completely abstract type that defines a contract — a set of methods that implementing classes must provide. All methods in an interface are public abstract by default (Java 7 and earlier). Java 8+ allows default and static methods.

  • Declared with interface keyword; implemented with implements.
  • A class can implement multiple interfaces (Java's way of multiple inheritance).
  • All fields in an interface are public static final by default.
  • An interface can extend another interface with extends.
Interfaces Example Java
interface Drawable {

}

interface Resizable {
    void resize(double factor);
}

// Implementing multiple interfaces
class Square implements Drawable, Resizable {
    double side;

    Square(double s) { side = s; }

    public void draw()                { System.out.println("Drawing square"); }
    public void resize(double factor) { side *= factor; }
}

        response.setContentType("text/html");
        PrintWriter out = response.getWriter();

        String name = request.getParameter("name");
        if (name == null) name = "World";

        out.println("<html><body>");
        out.println("<h1>Hello, " + name + "!</h1>");
        out.println("</body></html>");
    }

    // Handles HTTP POST requests
    protected void doPost(HttpServletRequest request,
                           HttpServletResponse response)
            throws ServletException, IOException {
        doGet(request, response); // delegate to doGet
    }

    // Called once when servlet is being removed
    public void destroy() {
        System.out.println("Servlet destroyed");
    }
}

import java.sql.*;

public class JDBCDemo {
    public static void main(String[] args) {

        Connection con = null;
        Statement stmt = null;
        ResultSet rs  = null;

        try {
            // STEP 1: Load and Register Driver
            Class.forName("com.mysql.cj.jdbc.Driver");
            System.out.println("Driver loaded.");

            // STEP 2: Establish Connection
            con = DriverManager.getConnection(
                    "jdbc:mysql://localhost:3306/college",
                    "root", "password");
            System.out.println("Connected to database.");

            // STEP 3: Create Statement
            stmt = con.createStatement();

            // STEP 4: Execute Query
            rs = stmt.executeQuery("SELECT * FROM students");

            // STEP 5: Process ResultSet
            while (rs.next()) {
                int id     = rs.getInt("student_id");
                String name = rs.getString("name");
                double marks = rs.getDouble("marks");
                System.out.printf("ID: %d | Name: %-15s | Marks: %.2f%n",
                                  id, name, marks);
            }

        } catch (ClassNotFoundException e) {
            System.out.println("Driver not found: " + e.getMessage());
        } catch (SQLException e) {
            System.out.println("SQL error: " + e.getMessage());
        } finally {
            // STEP 6: Close Connection (always in finally)
            try {
                if (rs   != null) rs.close();
                if (stmt != null) stmt.close();
                if (con  != null) con.close();
            } catch (SQLException e) { e.printStackTrace(); }
        }
    }
}

📚 Reference — Glossary

Java Glossary of Terms

Key definitions for all important Java and OOP terms — ideal for quick revision and viva voce preparation.

Abstract Class
A class declared with abstract keyword that cannot be instantiated. May contain abstract (no-body) and concrete methods. Used as a base template.
Abstract Method
A method declared without a body inside an abstract class. Subclasses must override and implement it.
Autoboxing
Automatic conversion of a primitive type (e.g., int) to its corresponding wrapper class object (e.g., Integer) by the Java compiler.
Bytecode
Platform-independent intermediate code produced by the Java compiler (javac) from .java source files. Stored in .class files and executed by the JVM.
Class
A user-defined blueprint or template that defines the structure (fields) and behavior (methods) of its objects. A logical entity.
Constructor
A special method with the same name as the class and no return type. Called automatically when an object is created using new. Initializes object state.
Dynamic Binding
The process of resolving a method call at runtime rather than compile time. Occurs with method overriding. Also called late binding.
Encapsulation
Wrapping data (fields) and code (methods) together in a class, and restricting direct access using access modifiers. Also called data hiding.
Exception
An abnormal event that disrupts normal program execution during runtime. Represented by objects derived from the Throwable class hierarchy.
final
A keyword that makes: variables constant, methods non-overridable, and classes non-extendable.
finally
A block in exception handling that always executes, whether or not an exception is thrown. Used for cleanup (closing files, connections).
Garbage Collection
Automatic memory management in Java. The JVM reclaims heap memory occupied by objects that are no longer reachable/referenced.
Inheritance
A mechanism where a subclass acquires fields and methods from a superclass using extends. Establishes an IS-A relationship and promotes code reuse.
Interface
A completely abstract type in Java that defines a contract of methods a class must implement. Declared with interface, implemented with implements. Supports multiple inheritance.
JDK
Java Development Kit. The complete Java development environment including JRE, compiler (javac), debugger, javadoc, and development tools.
JDBC
Java Database Connectivity. A Java API (java.sql package) that allows Java programs to connect to and execute SQL against relational databases.
JIT Compiler
Just-In-Time Compiler. Part of the JVM that compiles frequently executed bytecode to native machine code at runtime for improved performance.
JRE
Java Runtime Environment. Contains JVM plus standard class libraries required to run Java programs. Cannot be used to develop Java programs.
JSP
Java Server Pages. A server-side technology to create dynamic web pages by embedding Java code into HTML using special tags (<% %>, <%= %>, <%! %>).
JVM
Java Virtual Machine. An abstract computing machine that enables Java's platform independence by executing bytecode on any host system.
Message Passing
The mechanism by which objects communicate by calling each other's methods. Syntax: object.method(args).
Method Overloading
Defining multiple methods in the same class with the same name but different parameter lists. Resolved at compile time (static polymorphism).
Method Overriding
Redefining a parent class method in a subclass with the same signature. Resolved at runtime (dynamic polymorphism). Uses @Override annotation.
Multithreading
The ability to execute multiple threads concurrently within a single process. Threads share the process's heap memory but have their own stack.
Object
An instance of a class. A real-world entity with state (fields), behavior (methods), and identity (unique reference). Created with new. A physical entity.
Package
A namespace grouping related classes and interfaces. Prevents naming conflicts, controls access. Declared with package, imported with import.
Polymorphism
The ability of one interface or method to take multiple forms. "Many forms." Achieved via overloading (compile-time) and overriding (runtime).
PreparedStatement
A JDBC interface for executing pre-compiled, parameterized SQL queries. More secure (prevents SQL injection) and efficient than Statement for repeated queries.
Servlet
A Java class that runs on a web server, extends HttpServlet, and handles HTTP requests/responses. Has a lifecycle: init(), service(), destroy().
String
An immutable sequence of characters in Java. Every modification creates a new String object. Part of java.lang package (auto-imported).
StringBuffer
A mutable, thread-safe sequence of characters. Allows in-place modification. Preferred over String for repeated concatenation in multi-threaded environments.
super
A reference to the immediate parent class object. Used to call parent constructors (super()) and access overridden parent methods/fields (super.method()).
Synchronization
A mechanism to control access of multiple threads to shared resources. The synchronized keyword ensures only one thread executes a block at a time (mutual exclusion).
this
A reference to the current object. Used to distinguish instance variables from parameters (same name), call other constructors (this()), and pass current object as argument.
Thread
The smallest unit of execution within a process. Shares heap memory with other threads of the same process but has its own program counter and stack.
throw
A keyword used to explicitly throw an exception object from within a method body: throw new ExceptionType("message");
throws
A keyword in the method signature that declares which checked exceptions the method may throw, requiring the caller to handle or re-declare them.
Type Casting
Converting a value from one data type to another. Widening (implicit, safe) converts smaller to larger. Narrowing (explicit, may lose data) converts larger to smaller.
Wrapper Class
A class that wraps a primitive type in an object (e.g., Integer for int). Found in java.lang package. Essential for Collections and APIs requiring objects.
WORA
Write Once, Run Anywhere. Java's key feature: bytecode compiled once on any platform runs on any platform that has a compatible JVM.

  • Ternary: (cond) ? val1 : val2

Unit II — Control Flow & Classes

🔀 Decision & Loops

  • if, if-else, else-if, switch
  • for — iterations known; while — entry-controlled
  • do-while — exit-controlled (runs at least once)
  • break — exit loop; continue — skip iteration
  • switch needs break to prevent fall-through

🏛️ Constructors

  • Same name as class, no return type
  • Default (no args), Parameterized, Copy
  • this() calls another constructor (must be first)
  • If you define any constructor, default removed
  • Cannot be static, abstract, or final

🔒 Access Modifiers

  • public — everywhere
  • protected — package + subclasses
  • default — same package only
  • private — same class only
  • Most restrictive for fields → use private + getters

🆚 Overloading vs Overriding

  • Overloading: same class, same name, diff params
  • Overriding: child redefines parent's method exactly
  • Overloading → compile-time (static binding)
  • Overriding → runtime (dynamic binding)
  • @Override annotation — compiler validation

📝 String Key Methods

  • length(), charAt(i), indexOf(), substring()
  • toUpperCase(), toLowerCase(), trim()
  • equals() [not ==], contains(), replace()
  • split(), startsWith(), endsWith()
  • StringBuffer: append, insert, delete, reverse

Unit III — Inheritance, Threads & Exceptions

🌳 Inheritance Types

  • Single: A → B
  • Multilevel: A → B → C
  • Hierarchical: A → B, A → C
  • Multiple (via interfaces): B implements I1,I2
  • Java classes: single inheritance only
  • super() — calls parent constructor (first stmt)

🧵 Thread Lifecycle

  • New → Runnable → Running → Blocked/Waiting → Dead
  • start() → calls run() in new thread
  • sleep(ms) → blocked; join() → wait for thread
  • Priority: MIN=1, NORM=5, MAX=10
  • synchronized — mutual exclusion
  • Thread vs Runnable: Runnable preferred

⚠️ Exceptions Quick

  • Checked: must handle (IOException, SQLException)
  • Unchecked: optional (NullPointer, ArrayIndex...)
  • Error: JVM problems (OutOfMemory, StackOverflow)
  • throw — throws an exception object
  • throws — declares method may throw
  • finally — always executes

🔌 Abstract vs Interface

  • Abstract: partial implementation, extends once
  • Interface: full contract, implements many
  • Interface fields: public static final
  • Interface methods: public abstract (default)
  • Java 8+: default & static methods in interface

Unit IV — JSP, Servlets & JDBC

📄 JSP Scripting Elements

  • <%! %> — Declaration (class-level vars/methods)
  • <% %> — Scriptlet (per-request Java code)
  • <%= %> — Expression (prints to response)
  • <%-- --%> — JSP Comment
  • Lifecycle: Translation→Compile→init→service→destroy

🌐 Servlet Lifecycle

  • Load → Instantiate → init() → service() → destroy()
  • init() — once; destroy() — once; service() — per request
  • doGet() — HTTP GET; doPost() — HTTP POST
  • web.xml maps URL pattern to Servlet class
  • WEB-INF is protected (not browser-accessible)

🗄️ JDBC 6 Steps

  • 1. Load Driver: Class.forName("driver")
  • 2. Connect: DriverManager.getConnection(url,u,p)
  • 3. Statement: con.createStatement()
  • 4. Execute: executeQuery() / executeUpdate()
  • 5. ResultSet: rs.next(), rs.getString()
  • 6. Close: rs, stmt, con in finally block

📦 JSP Implicit Objects

  • request — HttpServletRequest
  • response — HttpServletResponse
  • out — JspWriter (write to browser)
  • session — HttpSession
  • application — ServletContext
  • exception — for error pages only

⚠️ Common Mistakes Students Make

Avoid These Mistakes
  • Using == for Strings: Always use .equals() for String comparison; == compares references.
  • Forgetting break in switch: Without break, execution falls through to the next case.
  • Calling run() instead of start(): t.run() runs in the current thread; t.start() creates a new thread.
  • Not closing JDBC resources: Always close ResultSet, Statement, Connection in a finally block.
  • Confusing throw and throws: throw is used inside a method; throws is in the method signature.
  • Using = instead of == in conditions: if (x = 5) is assignment; if (x == 5) is comparison.
  • Returning value from constructor: Constructors have no return type — not even void.
  • Instantiating abstract class: new AbstractClass() causes compile error.
  • Not using @Override: Without it, a typo in method name creates overloading, not overriding.
  • JSP Expression with semicolon: <%= value; %> is wrong; expressions have no semicolon.
