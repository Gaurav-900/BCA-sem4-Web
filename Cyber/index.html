1.1 Introduction to Cyber Space

Definition: Cyberspace is a global domain within the information environment consisting of interdependent networks of information technology infrastructure including the internet, telecommunications networks, computer systems, and embedded processors and controllers.

The term "cyberspace" was first coined by science-fiction author William Gibson in his 1982 short story Burning Chrome and popularized in his 1984 novel Neuromancer. Today, it refers to the virtual environment created by computers and networks where digital communication and interaction occur.

Key Characteristics of Cyberspace

  • Ubiquity: Accessible from anywhere in the world at any time
  • Anonymity: Users can mask their real identity
  • Borderlessness: No physical or geographic boundaries
  • Rapid Evolution: Technology and threats evolve at an unprecedented pace
  • Interconnectedness: All devices and systems are linked, making vulnerabilities contagious

Importance of Cyberspace

  • Enables global communication and commerce
  • Supports critical national infrastructure (power grids, banking, defence)
  • Platform for education, governance (e-governance), and healthcare
  • Source of information and entertainment

Components of Cyberspace

1.2 History of Internet

The Internet evolved from a military research network into the global infrastructure we depend on today. Understanding its history helps appreciate security challenges that emerged at each stage.

1969
ARPANET – Advanced Research Projects Agency Network created by US DoD. First message sent between UCLA and Stanford. First packet-switched network.
1971
Email Invented – Ray Tomlinson sends the first email using the @ symbol to separate username from host.
1973
TCP/IP Conceptualized – Vint Cerf and Bob Kahn design Transmission Control Protocol / Internet Protocol, the foundation of modern internet.
1983
DNS Created – Domain Name System introduced, replacing numeric IP addresses with memorable names.
1989
WWW Invented – Tim Berners-Lee proposes the World Wide Web at CERN, Switzerland.
1991
Internet Goes Public – WWW made publicly available. First web browser (Mosaic, 1993) makes the internet accessible to non-technical users.
1995–2000
Dot-com Boom – Explosive commercial growth. E-commerce, search engines (Google, 1998), and online banking emerge.
2004+
Web 2.0 / Social Media – User-generated content, social networks (Facebook 2004, YouTube 2005, Twitter 2006). New attack surfaces emerge.
2010+
Mobile Internet & Cloud – Smartphones become primary internet devices. Cloud computing, IoT, and Big Data reshape the landscape. India's internet revolution accelerates post-Jio (2016).
2020+
Web 3.0 / AI Era – Blockchain, metaverse concepts, AI-driven cyber attacks. 5G enables hyper-connectivity. India reaches 900M+ internet users.

Internet in India

  • 1995: VSNL (Videsh Sanchar Nigam Limited) launches public internet in India
  • 1998: Internet privatised; ISP licensing begins
  • 2016: Reliance Jio launch triggers explosive data consumption
  • 2022: India has world's second-largest internet user base (~850M users)
  • BharatNet: Government project to connect all gram panchayats via optical fiber

1.3 Cyber Crime

Definition: Cyber Crime refers to any criminal activity that involves a computer, networked device, or a network. Either the computer is the target of the crime, or it is used as a tool to commit the crime.

Classification of Cyber Crimes

CategoryDescriptionExamples
Against IndividualsCrimes targeting personal data, privacy, or reputationCyberstalking, identity theft, phishing, morphing images
Against PropertyUnauthorized access or destruction of digital propertyHacking, virus attacks, IPR theft, data theft
Against OrganizationsAttacks on corporate or government systemsRansomware, DDoS, corporate espionage, data breach
Against Society/StateCrimes threatening national security or public orderCyber terrorism, cyber warfare, illegal content distribution

Major Types of Cyber Crimes

🎣 Phishing

Fraudulent attempt to obtain sensitive information by disguising as a trustworthy entity via email, SMS (smishing), or voice calls (vishing).

🔒 Ransomware

Malicious software that encrypts victim's files and demands payment (ransom) for decryption key. Example: WannaCry (2017) affected 150+ countries.

👤 Identity Theft

Stealing personal information (Aadhaar, PAN, banking credentials) to commit fraud or impersonate the victim.

🌊 DDoS Attack

Distributed Denial of Service – Overwhelming a server with massive traffic from multiple sources to make it unavailable to legitimate users.

💸 Online Fraud

Financial fraud via fake e-commerce sites, lottery scams, job scams, OTP fraud, or fake investment schemes (Ponzi schemes online).

🕵 Cyber Stalking

Using the internet to harass, threaten, or monitor a person without their consent. Often targeting women and minors.

Cyber Crime vs Traditional Crime

EvidencePhysical evidenceDigital/electronic evidence
AnonymityDifficult to remain anonymousEasy to use fake identities
ScaleLimited geographic reachCan affect millions simultaneously
CostRequires physical resourcesLow cost; tools available online
JurisdictionClear jurisdictionComplex cross-border jurisdiction

1.4 Information Security

Definition: Information Security (InfoSec) is the practice of protecting information and information systems from unauthorized access, use, disclosure, disruption, modification, or destruction, in order to provide confidentiality, integrity, and availability.

The CIA Triad – Core Principles of Information Security

🔐 Confidentiality

Ensuring information is accessible only to those authorized to have access. Mechanisms: encryption, access controls, authentication.

Threat: Eavesdropping, data breach, insider threat

✅ Integrity

Ensuring information is accurate and has not been tampered with. Mechanisms: hashing (MD5, SHA), digital signatures, checksums.

Threat: Man-in-the-middle attack, data corruption

🌐 Availability

Ensuring authorized users can access information when needed. Mechanisms: redundancy, backup, load balancing, DDoS protection.

Threat: DDoS, ransomware, hardware failure

+ Additional Principles

Authentication: Verifying identity of users/systems

Non-repudiation: Ensuring parties cannot deny their actions

Authorization: Defining what an authenticated user can do

Types of Security Controls

TypeDescriptionExamples
PreventiveStop attacks before they happenFirewalls, encryption, strong passwords, 2FA
DetectiveIdentify attacks in progress or after they occurIDS, audit logs, antivirus alerts
CorrectiveRestore systems after an attackBackup restoration, patch management, incident response
DeterrentDiscourage potential attackersWarning banners, legal notices, CCTV

Security Policies

Definition: A Security Policy is a document that outlines rules, procedures, and guidelines for protecting an organization's information assets.

Key components of a security policy include:

  • Acceptable Use Policy (AUP): Rules on appropriate use of organizational IT resources
  • Access Control Policy: Who can access what resources and under what conditions
  • Password Policy: Rules for creating and managing passwords
  • Incident Response Policy: How to respond to security breaches
  • Data Classification Policy: Categorizing data by sensitivity (Public, Internal, Confidential, Secret)
  • BYOD Policy: Rules for personal devices used for work

1.5 Computer Ethics

Definition: Computer Ethics is a branch of applied ethics that examines ethical issues arising from the use of computers and information technology, including issues of privacy, intellectual property, professional responsibility, and computer crime.

Ten Commandments of Computer Ethics (Computer Ethics Institute)

  1. Thou shalt not use a computer to harm other people
  2. Thou shalt not interfere with other people's computer work
  3. Thou shalt not snoop around in other people's computer files
  4. Thou shalt not use a computer to steal
  5. Thou shalt not use a computer to bear false witness
  6. Thou shalt not copy or use proprietary software for which you have not paid
  7. Thou shalt not use other people's computer resources without authorization
  8. Thou shalt not appropriate other people's intellectual output
  9. Thou shalt think about the social consequences of the program you are writing
  10. Thou shalt always use a computer in ways that ensure consideration and respect for fellow humans

Key Areas of Computer Ethics

🔏 Privacy

Right to control personal information. Issues: data collection without consent, surveillance, cookies, profiling by tech companies. India's Digital Personal Data Protection Act 2023 addresses this.

©️ Intellectual Property

Software piracy, copyright infringement, plagiarism. India's Copyright Act 1957 and IT Act 2000 provide legal protections.

⚖️ Access & Digital Divide

Equal access to technology. The digital divide between urban/rural, rich/poor raises ethical questions about technology equity.

🤖 AI Ethics

Emerging issues: algorithmic bias, deepfakes, autonomous weapons, AI decision-making in healthcare and justice systems.

Antivirus Software

Definition: Antivirus software is a program designed to detect, prevent, and remove malicious software (malware) from computer systems.

How Antivirus Works:

  • Signature-based detection: Compares files against a database of known malware signatures
  • Heuristic analysis: Detects previously unknown viruses by examining code behaviour patterns
  • Behavioural monitoring: Watches programs in real-time for suspicious activity
  • Sandboxing: Runs suspicious files in an isolated environment to observe behavior
  • Cloud-based detection: Uses cloud intelligence to identify new threats instantly

Popular Antivirus Solutions: Windows Defender (free), Kaspersky, Norton, McAfee, Quick Heal (India), Avast, Bitdefender

1.6 Email Security & Securing Web Browser

Email Security

Email is the #1 vector for cyber attacks. More than 90% of cyber attacks begin with a phishing email.

Common Email Threats
  • Phishing: Fake emails impersonating banks, government agencies, or employers
  • Spear Phishing: Targeted phishing with personalized details
  • Whaling: Phishing targeting senior executives (CEOs, CFOs)
  • Email Spoofing: Forging sender's email address to appear legitimate
  • Malicious Attachments: Word documents, PDFs with embedded malware
  • Business Email Compromise (BEC): Impersonating executives to authorize fraudulent payments
Email Security Best Practices
  • Never click links in unexpected emails – go directly to the website
  • Verify sender's actual email address (not just display name)
  • Never open unexpected attachments, especially .exe, .zip, .doc with macros
  • Enable spam filters and email scanning
  • Use S/MIME or PGP for encrypted email communications
  • Enable SPF, DKIM, and DMARC for organizational email security
  • Use separate email for personal vs official communications

Securing Web Browser

Browser Security Best Practices
  • Keep browser updated to latest version (patches security vulnerabilities)
  • Always check for HTTPS (padlock icon) before entering sensitive data
  • Review and restrict browser permissions (camera, microphone, location)
  • Use browser extensions judiciously – only from trusted sources
  • Enable pop-up blocker and phishing protection
  • Clear browsing history, cookies, and cache regularly
  • Disable AutoFill for passwords on shared computers
  • Use private/incognito mode on shared devices
  • Install ad-blockers to prevent malicious advertisements (malvertising)
  • Avoid saving passwords in browser (use dedicated password manager instead)
Understanding HTTPS
HTTPS (HyperText Transfer Protocol Secure) uses SSL/TLS encryption to secure communication between browser and server. Look for the padlock 🔒 icon. HTTP transmits data in plaintext – visible to attackers on same network.

1.7 Guidelines for Secure Password & Wi-Fi Security

Why Password Security Matters

Weak passwords are the most common entry point for cyber attacks. A simple 6-character numeric password can be cracked in milliseconds using modern computers. Strong passwords are the first line of defence.

Guidelines for Setting Up a Secure Password

✅ DO
  • Use at least 12–16 characters
  • Mix uppercase, lowercase, numbers, and special characters
  • Use a passphrase: "MyDog@Eats#3Bones!" is strong and memorable
  • Use a different password for every account
  • Change passwords periodically (every 90 days for critical accounts)
  • Use a password manager to generate and store passwords
❌ DON'T
  • Use obvious info: name, DOB, phone number
  • Use common passwords: "password", "123456", "qwerty"
  • Reuse the same password across multiple sites
  • Share passwords with anyone
  • Write passwords on paper near your computer
  • Use single words, even complex ones

Password Strength Examples

PasswordStrengthCrack Time (approx)
123456Very WeakInstantly
RameshWeakMilliseconds
Ram@1234MediumMinutes-Hours
R@m3sh#Jaipur!StrongYears
K#9pL$mN2@qR7!vXVery StrongCenturies

Wi-Fi Security Guidelines

  • Use WPA3 encryption (or minimum WPA2). Never use WEP (easily cracked)
  • Change default router admin username and password immediately
  • Use a strong, unique Wi-Fi password
  • Hide your SSID (network name) from broadcast
  • Enable MAC address filtering (whitelist known devices)
  • Disable remote management and WPS (Wi-Fi Protected Setup) – WPS has known vulnerabilities
  • Create a separate guest network for visitors
  • Keep router firmware updated
  • Public Wi-Fi warnings: Never access banking or sensitive sites on public Wi-Fi without VPN
  • Use VPN (Virtual Private Network) on public networks to encrypt traffic
WPA3 – Wi-Fi Protected Access 3 – Latest Wi-Fi security standard. Uses SAE (Simultaneous Authentication of Equals) – provides stronger protection against offline password guessing attacks compared to WPA2.

1.8 Two-Step Authentication & Password Manager

Two-Factor / Two-Step Authentication (2FA/2SA)

Definition: Two-Factor Authentication (2FA) is a security process in which users provide two different authentication factors to verify their identity, providing an additional layer of security beyond just a password.

Authentication Factors

Something You Know

Password, PIN, security question answers

Something You Have

OTP via SMS/email, hardware token (RSA SecurID), authenticator app (Google Authenticator)

Something You Are

Biometrics – fingerprint, face scan, iris scan, voice recognition

How 2FA Works

Types of 2FA Methods

MethodDescriptionSecurity Level
SMS OTPOne-time password via text messageMedium (SIM swap risk)
Email OTPOne-time code via emailMedium
TOTP AppTime-based OTP via Google Authenticator/AuthyHigh
Hardware TokenPhysical device generating codes (e.g., YubiKey)Very High
BiometricFingerprint or face recognitionHigh
Push NotificationApprove/deny login on trusted deviceHigh

Password Manager

Definition: A Password Manager is a software application that stores and manages passwords for various online accounts in an encrypted vault, accessible through a single master password.
Benefits of Password Managers
  • Generates strong, unique passwords for every site
  • Stores all passwords in an encrypted vault
  • Auto-fills credentials, reducing phishing risk
  • Alerts when passwords are found in data breaches
  • Syncs across devices
  • Eliminates password reuse (the #1 password security mistake)
Popular Password Managers

LastPass, Bitwarden (open source), 1Password, Dashlane, KeePass (offline), browser-built-in managers (basic). For India-based users, built-in managers in Chrome/Firefox are common starting points.

2.1 Guidelines for Basic Windows Security

Why Windows Security Matters

Windows is the world's most widely used operating system, making it the primary target for cyber attacks. A poorly configured Windows system is highly vulnerable to malware, ransomware, and unauthorized access.

Essential Windows Security Guidelines

🔄 Keep Windows Updated
  • Enable automatic Windows Updates
  • Install security patches immediately
  • Do not use outdated versions (XP, Vista – no longer supported)
🛡 Windows Defender
  • Built-in antivirus/antimalware for Windows 10/11
  • Keep real-time protection enabled
  • Run regular full scans
👤 User Account Control (UAC)
  • Never use Administrator account for daily tasks
  • Create a Standard user account for everyday use
  • UAC prompts provide a security checkpoint before system changes
🔥 Windows Firewall
  • Keep Windows Firewall enabled at all times
  • Review inbound/outbound rules regularly
  • Block unknown programs from network access
💾 Data Backup
  • Use Windows Backup or third-party tools
  • Follow 3-2-1 rule: 3 copies, 2 media types, 1 offsite
  • Back up to external drive and cloud (OneDrive, Google Drive)
🔐 BitLocker Encryption
  • Encrypt hard drives with BitLocker (Windows Pro/Enterprise)
  • Prevents data access if laptop is stolen
  • Store recovery key safely

User Account Password Security (Windows)

  • Set a strong login password or PIN for Windows account
  • Enable screen lock that activates after 2–5 minutes of inactivity
  • Use Windows Hello (biometric authentication) where available
  • Disable Guest account
  • Limit number of administrator accounts
  • Review and remove unused user accounts periodically

Additional Windows Hardening Tips

  • Disable AutoPlay/AutoRun for USB drives (prevents BadUSB attacks)
  • Use BitLocker to encrypt drives
  • Enable Secure Boot in BIOS/UEFI settings
  • Uninstall unnecessary programs and disable unused services
  • Configure browser to block malicious downloads
  • Enable SmartScreen filter

2.2 Social Media Security & Safe Social Networking

Social Media Security Risks

  • Account hijacking: Attackers gain unauthorized access to accounts
  • Identity theft: Personal information harvested from profiles
  • Catfishing: Fake profiles to deceive users
  • Social engineering: Manipulating users through social platforms
  • Malicious links: Spam posts with malware links
  • Privacy violations: Oversharing personal data
  • Reputation damage: Fake news, morphed images

Social Media Security Guidelines

✅ Best Practices
  • Use strong, unique password for each social platform
  • Enable 2FA on all social accounts
  • Review privacy settings – limit profile visibility
  • Regularly review which apps have access to your account
  • Log out from devices you don't own
  • Think before you post – content lives forever online
  • Report suspicious profiles and fake accounts
  • Verify information before sharing – avoid fake news
❌ Avoid
  • Don't share home address, phone number, or travel plans publicly
  • Don't accept friend requests from strangers
  • Don't click suspicious links even from friends (account may be hacked)
  • Don't participate in quizzes that request profile access
  • Don't post photos showing valuable items, documents, or location details
  • Don't share Aadhaar, PAN, or bank details anywhere on social media

Tips for Safer Social Networking

  • Privacy Audit: Review privacy settings quarterly on Facebook, Instagram, WhatsApp, Twitter
  • Trusted Contacts: Only connect with people you know in real life
  • Geolocation: Disable location tagging in photos and posts
  • App Permissions: Regularly audit which third-party apps can access your social accounts
  • Secure Devices: Keep phone and computer protected where social apps are installed
  • Breach Monitoring: Use haveibeenpwned.com to check if your email appears in known data breaches

2.3 Smartphone Security Guidelines

Why Smartphone Security is Critical

In India, most people access the internet primarily through smartphones. Smartphones hold banking apps, personal photos, Aadhaar details, UPI apps, and private messages – making them prime targets.

Smartphone Security Guidelines

  • Screen Lock: Always use PIN, password, pattern, or biometric lock
  • OS Updates: Keep Android/iOS updated – patches fix security vulnerabilities
  • App Sources: Only install apps from Google Play Store or Apple App Store
  • App Permissions: Review and restrict app permissions (camera, microphone, contacts, location)
  • Remote Wipe: Enable "Find My Phone" / "Find My Device" for remote data wipe if lost
  • Encryption: Enable full-device encryption (most modern phones do this by default)
  • Unknown Sources: Disable "Install from Unknown Sources" on Android
  • Backup: Regularly backup phone data to secure cloud storage
  • Bluetooth/NFC: Turn off Bluetooth and NFC when not in use
  • Public USB charging: Avoid "juice jacking" – do not charge phone from unknown USB ports
  • VPN: Use VPN when on public Wi-Fi
  • Antivirus: Install mobile security app (especially on Android)
Juice Jacking: A cyber attack where malware is installed, or data is stolen from a device through a compromised USB charging port or cable. Always carry your own charger.

2.4 Online Banking & Mobile Banking Security

Online Banking Security Threats

  • Phishing websites: Fake bank websites that steal credentials
  • Keyloggers: Malware that records keystrokes including passwords and OTPs
  • Man-in-the-Browser: Malware that intercepts banking transactions
  • SIM Swap: Fraudsters port your mobile number to steal OTPs
  • Vishing (Voice Phishing): Fake bank executives calling to get sensitive details
  • Fake Apps: Malicious apps mimicking legitimate banking apps

Online Banking Security Guidelines

  • Always type bank URL directly – never use links from emails or SMS
  • Verify the website has HTTPS and the correct domain name
  • Never access banking from public computers or public Wi-Fi
  • Register for SMS/email alerts for all transactions
  • Set transaction limits (daily, per-transaction limits)
  • Regularly review account statements for unauthorized transactions
  • Never share OTP, CVV, PIN, or full card number with anyone – including bank employees
  • Log out completely after every banking session
  • Clear browser cache after banking session on shared computers

Mobile Banking Security

  • Download banking apps only from official bank websites or trusted stores
  • Verify app publisher details before installing
  • Enable app-specific lock (fingerprint/PIN)
  • Never allow mobile banking apps to run on rooted/jailbroken phones
  • Disable screenshot capture within banking apps (many banks do this by default)
  • Report lost/stolen phone to bank immediately to block mobile banking
  • Enable notifications for all transactions

2.5 Credit Card, Debit Card & UPI Security

Card Security Basics

Card Skimming: Illegal copying of card data using a device placed on ATM card slots or POS terminals. Skimmers capture the magnetic stripe data which is then used to clone cards.

Security of Debit and Credit Cards

  • Memorize your PIN – never write it on the card or carry it in wallet
  • Shield keypad when entering PIN at ATM or POS
  • Inspect ATM card slot for skimming devices before inserting card
  • Enable SMS/email alerts for every transaction
  • Set international usage to "disabled" unless travelling abroad
  • For online shopping, use virtual cards or one-time use card numbers
  • Do not share full card number, CVV, and expiry date together (CVV should never be shared)
  • Report suspicious transactions immediately to bank
  • Use RuPay/VISA Secure / Mastercard SecureCode (3D Secure) for online transactions
  • Block card immediately if lost or stolen – use banking app or call bank helpline

UPI (Unified Payments Interface) Security

UPI is a real-time payment system developed by NPCI (National Payments Corporation of India) that enables instant fund transfers between bank accounts through mobile platforms. Apps: PhonePe, Google Pay, Paytm, BHIM.
  • Never share UPI PIN with anyone – not even bank officials
  • Verify recipient VPA (Virtual Payment Address) before sending money
  • Be cautious of QR code fraud – scanning a QR can debit money from your account (collect request disguised as payment)
  • Only ENTER PIN to send money; a PIN is NEVER needed to receive money
  • Report fraudulent UPI transactions on NPCI portal or call 1800-120-1740
  • Use only official UPI apps
  • Enable app lock on UPI applications
⚠️ UPI Scam Alert: If someone asks you to scan a QR code or enter your UPI PIN to "receive" money, it is a SCAM. You never need to enter a PIN or scan QR to receive money on UPI.

POS (Point of Sale) Security

  • Use chip-based cards (EMV) rather than swipe (magnetic stripe)
  • Prefer contactless payments (tap) for small amounts – reduces skimming risk
  • Do not let cashier take card out of your sight
  • Check POS receipt and match amount before entering PIN
  • Cover PIN keypad while entering

Security of Micro ATMs

Micro ATM: A handheld device used by Business Correspondents (BCs) in rural India for Aadhaar-enabled banking transactions. It connects to the bank's core system via the internet.
  • Only use Micro ATMs operated by RBI-certified Business Correspondents
  • Verify the BC's identity and certification
  • Ensure biometric (fingerprint) authentication is used
  • Get printed receipt for every transaction
  • Check SMS transaction alert matches receipt

2.6 e-Wallet Security Guidelines

e-Wallet (Digital Wallet): A software-based system that stores users' payment information and passwords for numerous payment methods and websites. Examples: Paytm, Amazon Pay, MobiKwik, PhonePe wallet, Google Pay balance.

e-Wallet Security Guidelines

  • Use only RBI-registered payment wallets
  • Enable strong password and biometric lock on wallet apps
  • Do not link multiple bank accounts to wallet unless necessary
  • Keep wallet balance minimal – only top up when needed
  • Enable transaction SMS/email alerts
  • Regularly review transaction history for unauthorized activity
  • Do not store card details on unverified websites or wallets
  • Report unauthorized transactions immediately to wallet provider and RBI (Ombudsman: rbi.org.in/cms/Scripts/BS_OmbudsmanList.aspx)
  • Keep KYC details updated – full KYC provides higher transaction limits and better security
  • Enable device binding – wallet should only work on your registered device

3.1 Social Engineering

Definition: Social Engineering is the psychological manipulation of people into performing actions or divulging confidential information. It exploits human psychology rather than technical vulnerabilities. It is often described as "hacking the human" rather than hacking systems.

Why Social Engineering Works – Psychology Behind It

👔 Authority

People comply with requests from perceived authority figures (police, bank officials, IT department heads, government agencies).

⏰ Urgency & Scarcity

Creating time pressure ("Your account will be blocked in 2 hours!") forces hasty decisions without rational thinking.

🤝 Reciprocity

Offering something free creates a sense of obligation. "We've given you ₹500 cashback – just verify your account."

😰 Fear

"Your computer is infected – call now!" Fear-based messages bypass rational judgment and trigger immediate compliance.

💞 Liking

We comply with people we like. Attackers build rapport before making malicious requests.

📊 Social Proof

"Everyone is using this! Don't miss out." Leverages herd mentality to validate fraudulent schemes.

3.3 How Cyber Criminals Work & Prevention

Cyber Criminal Ecosystem

Modern cybercrime is organized and operates like a business, with specialized roles:

  • Exploit Developers: Create tools to exploit software vulnerabilities
  • Malware Authors: Write malicious software (viruses, ransomware, spyware)
  • Botnet Operators: Control networks of infected computers
  • Money Mules: Transfer stolen money to avoid tracing
  • Dark Web Merchants: Buy and sell stolen data, credentials, and hacking tools

Crime-as-a-Service (CaaS)

The dark web has created a marketplace where cyber crime tools are sold as services. Ransomware-as-a-Service (RaaS) allows non-technical criminals to deploy ransomware for a share of profits. Phishing kits cost as little as ₹500 on dark web forums.

How to Prevent Being a Victim of Cyber Crime

Technical Measures
  • Keep all software updated
  • Use antivirus and firewall
  • Enable 2FA on all accounts
  • Use strong, unique passwords
  • Encrypt sensitive data
  • Regular backups
Behavioural Measures
  • Be skeptical of unsolicited calls/emails
  • Verify identity before sharing information
  • Never share OTP with anyone
  • Report suspicious activity immediately
  • Educate family members, especially elderly and children

3.5 Cyber Security Techniques

Key Cyber Security Techniques

1. Encryption
Encryption is the process of converting plaintext into unreadable ciphertext using an algorithm and key, so that only authorized parties with the correct key can decrypt and read it.
  • Symmetric encryption: Same key for encryption and decryption (AES, DES). Fast but key distribution is a challenge.
  • Asymmetric encryption: Public key encrypts, private key decrypts (RSA, ECC). Used in SSL/TLS, digital signatures.
  • Hashing: One-way transformation (MD5, SHA-256). Used for password storage and integrity verification.
2. Intrusion Detection System (IDS) & Prevention System (IPS)
IDS monitors network/system activity for malicious activities and reports them. IPS goes further – it automatically blocks detected threats.
FeatureIDSIPS
ActionDetects and alertsDetects and blocks
PlacementOut-of-band (monitoring)Inline (active traffic path)
ResponsePassiveActive
3. VPN (Virtual Private Network)
VPN creates an encrypted tunnel between the user's device and a VPN server, masking the user's IP address and encrypting all internet traffic. Essential for public Wi-Fi use and remote work.
4. PKI (Public Key Infrastructure)

Framework for managing digital certificates and encryption keys. Enables secure HTTPS, digital signatures, and authentication. Components: Certificate Authority (CA), Registration Authority (RA), digital certificates.

5. Digital Forensics

Process of collecting, preserving, analyzing, and presenting digital evidence for legal proceedings. Used in cyber crime investigation.

3.6 Firewall

Definition: A Firewall is a network security device (hardware or software) that monitors and controls incoming and outgoing network traffic based on predetermined security rules. It establishes a barrier between trusted internal networks and untrusted external networks (like the Internet).

How a Firewall Works

Types of Firewalls

TypeDescriptionOSI LayerPros/Cons
Packet Filtering FirewallInspects individual packets based on IP, port, protocol rulesNetwork (Layer 3)Fast; does not inspect content
Stateful Inspection FirewallTracks state of connections; more intelligent filteringTransport (Layer 4)Better security; moderate speed
Application Layer Firewall (Proxy)Inspects application-layer data (HTTP, FTP, DNS)Application (Layer 7)Deep inspection; slower
Next-Generation Firewall (NGFW)Combines traditional firewall with IPS, deep packet inspection, app awarenessAll layersMost comprehensive; expensive
Cloud Firewall (FWaaS)Firewall delivered as a cloud serviceAll layersScalable; no hardware needed

Firewall Rules

Firewall rules define what traffic is allowed or denied. Rules are processed in order (top to bottom). Key rule components:

  • Source IP/port: Where traffic originates
  • Destination IP/port: Where traffic is going
  • Protocol: TCP, UDP, ICMP
  • Action: Allow, Deny, or Log

Default Deny Rule: Best practice – block all traffic by default, then only allow what is explicitly permitted ("whitelist" approach).

4.2 Cyber Security Incident Handling & Assurance

Cyber Security Incident: Any event that has the potential to adversely affect the confidentiality, integrity, or availability of an organization's information assets.

Incident Response Lifecycle (NIST Framework)

Phases Explained

  1. Preparation: Develop policies, train staff, set up monitoring tools, create incident response team
  2. Detection & Analysis: Identify incident indicators (IDS alerts, user reports, antivirus alerts); assess severity
  3. Containment: Stop the spread of the incident; isolate affected systems; preserve evidence
  4. Eradication: Remove malware, close vulnerabilities, patch systems
  5. Recovery: Restore systems from clean backups; verify functionality; monitor for recurrence
  6. Post-Incident Activity: Document lessons learned; update security policies; improve defenses

Cyber Security Assurance

Cyber Security Assurance is the process of providing confidence that security measures are properly implemented and effective. It involves audits, testing, and compliance verification.
  • Vulnerability Assessment: Systematic review of security weaknesses in a system
  • Penetration Testing (Pen Testing): Authorized simulated cyber attack to find exploitable vulnerabilities
  • Security Audit: Systematic evaluation of an organization's information system against a set of criteria
  • Security Certification: ISO 27001 (ISMS), SOC 2, PCI DSS (for payment systems)
  • Red Team/Blue Team: Red team attacks; Blue team defends. Exercises to improve security posture

SQL Injection (SQLi) – Key Web Attack

SQL Injection: An attack where malicious SQL statements are inserted into input fields, causing the database to execute unintended commands.

Example: Username field input: ' OR '1'='1
This turns the SQL query into: SELECT * FROM users WHERE username='' OR '1'='1'
Since '1'='1' is always true, the attacker bypasses authentication.

Cross-Site Scripting (XSS)

XSS: Attacker injects malicious scripts into web pages viewed by other users. The script runs in the victim's browser, potentially stealing session cookies, redirecting users, or logging keystrokes.

Digital Infrastructure Security

  • Network Segmentation: Dividing network into zones (DMZ, internal, management) to limit attack spread
  • DMZ (Demilitarized Zone): Network segment containing public-facing servers, separated from internal network
  • Zero Trust Architecture: "Never trust, always verify" – no implicit trust even inside the network perimeter
  • CDN Security: Content Delivery Networks can absorb DDoS attacks
  • WAF (Web Application Firewall): Filters HTTP traffic to protect web applications

4.6 Defensive Programming

Defensive Programming is a software development approach where programmers assume that errors, invalid inputs, and hostile environments are inevitable, and write code that anticipates and handles these situations gracefully while maintaining security.

Principles of Defensive Programming

  • Input Validation: Never trust user input. Validate all input on the server side. Use whitelists not blacklists.
  • Least Privilege: Programs should run with minimum required permissions
  • Fail Securely: When a system fails, it should do so in a secure state (e.g., deny access rather than allow)
  • Defense in Depth: Multiple security layers – if one fails, others remain
  • Avoid Security by Obscurity: Security should not depend on secrecy of design (algorithms, source code)
  • Separation of Privilege: Critical operations require multiple conditions to be satisfied
  • Error Handling: Do not expose system information in error messages
  • Parameterized Queries: Use prepared statements to prevent SQL injection
  • Output Encoding: Encode output to prevent XSS attacks
  • Secure Defaults: Systems should be secure by default, with security features enabled out-of-the-box

Secure Coding Practices

❌ Vulnerable Code (SQL Injection Risk)
query = "SELECT * FROM users WHERE username='" + username + "' AND password='" + password + "'"
✅ Secure Code (Parameterized Query)
query = "SELECT * FROM users WHERE username=? AND password=?"
cursor.execute(query, (username, password))

Security Testing Methods

  • SAST (Static Application Security Testing): Analyzes source code without executing it to find vulnerabilities
  • DAST (Dynamic Application Security Testing): Tests running application by simulating external attacks
  • IAST (Interactive AST): Combines SAST and DAST; tests from inside the running application
  • Fuzzing: Feeding random/unexpected inputs to find crashes and vulnerabilities
  • Code Review: Manual inspection of source code for security flaws

⚠️ Common Exam Mistakes to Avoid

  • Don't confuse IDS (detect only) with IPS (detect + block)
  • Don't say "WPA2 is the latest" – it's WPA3 (since 2018)
  • Section 66A was struck down by Supreme Court in Shreya Singhal case 2015
  • UPI PIN is needed to send money; never to receive
  • CVV should never be stored or shared – not even with bank employees
  • Worm = self-replicates without human action; Virus needs a host program
  • Phishing ≠ Spear Phishing – Spear is targeted with personal info
  • CERT-In was established in 2004, not 2000 (IT Act was 2000)
  • DPDP Act was enacted in 2023 (not 2020 or 2022)
  • Don't confuse encryption (reversible) with hashing (one-way, irreversible)
